Detection engine built for production

Every signal is captured with multiple verification paths, so detection stays reliable as the web evolves.

8 detection vectors

HTML, response headers, script sources, CSS rules, meta tags, JS globals, cookies, and URL patterns — combined with confidence scoring.

Contact email extraction

mailto links, plaintext patterns, JSON-LD Organization.email, data-* attributes, and SSR hydration data (__NEXT_DATA__, __NUXT__).

Contact page waterfall

If the main page has no email, fall back to /contact, /kontak, /redaksi via parallel HTTP + headless navigation.

Multi-tenant auth

API keys, HTTP Basic, and JWT (HS256) with per-user rate limits and usage tracking. Bootstrap admin from env vars at first run.

Persistent cache

24h TTL with disk persistence (data/cache.json) — repeated scans finish instantly without re-hitting the target.

UA rotation & stealth

14 real browser User-Agents (Chrome/Firefox/Safari/Edge, Win/Mac/Linux) — bypasses naive bot detection. JS __proto__ walking for minified SDKs.

18 top-level category groups

Raw detections roll up into priority-ordered groups so results are scannable, not a flat dump.

Up and running in 60 seconds

Self-hosted, license-clean, no telemetry. Just Go and a browser.

1. Build
$ git clone https://github.com/dikdotcom/stackly.git
$ cd stackly && go build -o stackly ./cmd/server
2. Run
$ STACKLY_BASIC_USERS="admin:changeme" ./stackly
→ listening on :8890
3. Scan
$ curl -u admin:changeme -X POST http://localhost:8890/api/scan \
  -H "Content-Type: application/json" \
  -d '{"url": "https://react.dev"}'
→ {"id":"a1b2c3","status":"pending","url":"https://react.dev"}
